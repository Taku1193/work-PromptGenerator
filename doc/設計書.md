# AI相談プロンプト生成ツール 設計書

## 1. 目的

業務中のバグ修正・不具合調査について、GitHub Copilot や社内利用可能なAIに相談しやすいプロンプトを、ローカル環境で素早く生成する。

本ツールは以下を満たす。

- Windows 上で動作する
- 外部API通信を行わない
- 入力内容をテンプレートへ整形して出力する
- 将来的な作業種別追加に対応しやすい

## 2. スコープ

今回のMVPでは以下を対象とする。

- 作業種別は `バグ修正` を初期実装とする
- GUI で入力し、生成結果を画面表示する
- 生成結果をクリップボードへコピーできる
- 入力内容と出力内容をクリアできる

今回のMVPでは以下は対象外とする。

- AI APIとの直接連携
- 外部通信
- DB保存
- 入力履歴の管理
- 複数テンプレートの動的読込

## 3. 想定利用者

- 保守開発を行う開発者
- 不具合調査を行う運用担当者
- AIへ相談する前に情報を整理したい担当者

## 4. 利用シナリオ

1. ユーザーが作業種別、相談内容、対象、発生事象などをGUIへ入力する
2. ユーザーが `プロンプト生成` ボタンを押す
3. ツールが入力値をテンプレートへ流し込み、構造化された相談文を生成する
4. ユーザーが生成結果を確認し、`コピー` ボタンでクリップボードへコピーする
5. ユーザーがGitHub Copilotや社内AIへ貼り付けて相談する

## 5. 機能要件

### 5.1 入力項目

画面には以下の入力欄を持つ。

1. 作業種別
2. 相談内容
3. 対象
4. 関連ソース・場所
5. 発生事象
6. 期待する動作
7. 実際の動作
8. エラー内容・ログ
9. 現状わかっていること
10. 制約・注意点
11. 聞きたいこと・やってほしいこと

### 5.2 ボタン機能

- `プロンプト生成`
  - 入力内容をもとにプロンプトを生成する
- `コピー`
  - 生成結果をクリップボードにコピーする
- `クリア`
  - 入力欄と出力欄を初期化する

### 5.3 出力要件

生成結果は以下の構造を持つ。

- 役割
- 作業種別
- 相談内容
- 対象
- 関連ソース・場所
- 発生事象
- 期待する動作
- 実際の動作
- エラー内容・ログ
- 現状わかっていること
- 制約・注意点
- 依頼したいこと
- 出力形式

未入力項目は見出しを含めて出力しない。

## 6. 非機能要件

- Python で実装する
- GUI は日本語 IME との相性を優先し、標準 `tkinter` / `ttk` を使用する
- 小規模構成で管理しつつ、`src/` と `doc/` を基準に整理する
- 外部ネットワーク通信を行わない
- Windows 実行を前提とする
- 後から作業種別を追加しやすい構成とする

## 7. 画面設計

### 7.1 画面構成

画面は大きく以下の2ペイン構成とする。

- 左ペイン: 入力エリア
- 右ペイン: 生成結果表示エリア

### 7.2 入力エリア

- スクロール可能なフォームとする
- 各項目はラベル + 入力欄で構成する
- 作業種別は将来の選択式対応を見据えてプルダウン形式とする
- 複数行入力を想定する項目はテキストボックスとする

### 7.3 出力エリア

- 生成されたプロンプト全文を複数行テキストボックスへ表示する
- 初期状態では案内文を表示する
- ステータスメッセージで操作結果を表示する

## 8. システム構成

### 8.1 構成方針

現時点では以下の構成とする。

- `app.py`
  - 起動用エントリポイント
- `src/`
  - GUI 本体
  - テンプレート定義
  - プロンプト生成処理
- `doc/`
  - 設計書
  - 要件メモ
- `tests/`
  - テンプレートの基本テスト
- `requirements.txt`
  - 依存ライブラリ定義

### 8.2 実行方式

- ローカルPC上で `python app.py` を実行する
- ネットワーク通信は発生しない

## 9. 内部設計

### 9.1 データ構造

テンプレートの拡張性を優先し、以下の構造を採用する。

- `FieldDefinition`
  - 画面項目定義
  - 項目キー、表示名、入力種別、高さ、初期値を保持する
- `PromptTemplate`
  - 作業種別ごとのテンプレート定義
  - 役割文、出力形式、項目一覧を保持する
  - 入力値から最終プロンプトを組み立てる
- `TEMPLATES`
  - 作業種別名をキーとしたテンプレート管理辞書

### 9.2 画面制御

`PromptGeneratorApp` クラスが以下を担当する。

- メインウィンドウ生成
- 入力欄の生成
- 出力欄の生成
- ボタンイベント処理
- クリップボード操作
- ステータス表示

### 9.3 生成処理

1. 現在の入力欄の値を収集する
2. 選択中の作業種別に対応するテンプレートを取得する
3. 各項目をテンプレート順に整形する
4. 未入力値は見出しごと生成結果から除外する
5. 生成結果を右ペインへ表示する

## 10. 拡張方針

将来的な作業種別追加は以下の方針で対応する。

- `src/prompt_templates.py` に `PromptTemplate` を追加する
- `TEMPLATES` 辞書へ登録する
- 必要に応じて出力形式や役割文を作業種別ごとに変更する
- 入力項目差分が必要になった場合は、テンプレートごとに `fields` を差し替える

追加候補は以下を想定する。

- ログ分析
- ソース解説
- 設計書解説
- テスト観点作成
- JIRAチケット作成
- 調査報告作成

## 11. エラーハンドリング方針

- 生成前に未入力があってもエラーにはしない
- コピー対象が未生成の場合は警告ダイアログを表示する
- 想定外例外の高度な復旧処理はMVPでは実装しない

## 12. テスト観点

### 12.1 画面表示

- アプリが起動できること
- 入力欄、出力欄、各ボタンが表示されること
- 長文入力時にスクロールできること

### 12.2 生成機能

- 全項目入力時に所定フォーマットで出力されること
- 一部未入力時に対象項目の見出しと本文が出力されないこと
- 作業種別が正しく反映されること

### 12.3 コピー機能

- 生成後にクリップボードへコピーできること
- 未生成時に警告ダイアログが表示されること

### 12.4 クリア機能

- 入力欄が初期値へ戻ること
- 出力欄が初期表示へ戻ること
